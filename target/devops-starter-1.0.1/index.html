Request Body and Response Formats


(1) User format (don’t provide ID, for POST) { “id”: long, “name”: String, “lastName”: String, “email”: String, “” }
(2) User format (don’t provide ID, for POST) simpler format: { “name”: string, “city”: string, “phone”: string }
(3) User format (don’t provide ID, for POST) simpler format: { “name”: string, “street”: string, “city”: string, “phone”: string }
Errors (e) All errors are reported using this format (with the HTTP-status code matching the number) { status : statusCode, "msg": "Explains the problem" }
(e1) :{ status : 404, "msg": "Person with given id does not exist" }
(e2) :{ status : 404, "msg": "No persons with given hobby exist" }
(e3) :{ status : 404, "msg": "No persons found with given ZIP-code" }
(e4) :{ status : 404, "msg": " Given hobby does not exist" }
(e5) :{ status : 404, "msg": "Given hobby does not exist" }
(e6) :{ status : 400, "msg": "Field xxx is required" }
